# 周报助手存储规范

## 存储位置

所有周报记录存储在：
```
~/.claude/projects/-Users-forxia/memory/weekly-records/
```

## 文件结构

### 目录结构
```
weekly-records/
├── 2026-04/
│   ├── 2026-04-21.json
│   ├── 2026-04-22.json
│   ├── 2026-04-23.json
│   └── ...
├── 2026-05/
│   └── ...
└── index.json          # 索引文件，记录所有日期
```

### 单条记录文件

**文件命名**：`YYYY-MM-DD.json`

**文件内容**：
```json
{
  "date": "2026-04-27",
  "weekday": "周一",
  "records": [
    {
      "timestamp": "2026-04-27T10:30:00",
      "content": "测试在线客服工具，修复3个错误，准确率提升至92%"
    }
  ]
}
```

**注意**：
- 同一天多次输入，追加到 `records` 数组
- `timestamp` 精确到秒，用于判断追加顺序

### 索引文件

**文件路径**：`weekly-records/index.json`

**文件内容**：
```json
{
  "last_updated": "2026-04-27T16:30:00",
  "records": {
    "2026-04-21": { "weekday": "周一", "count": 1 },
    "2026-04-22": { "weekday": "周二", "count": 1 },
    "2026-04-23": { "weekday": "周三", "count": 2 }
  }
}
```

## 操作规则

### 记录模式（今日总结：）

1. 解析输入，提取日期（默认当天）
2. 检查 `weekly-records/YYYY-MM/` 目录是否存在，不存在则创建
3. 检查 `YYYY-MM-DD.json` 是否存在
   - **存在**：读取文件，将新内容追加到 `records` 数组
   - **不存在**：创建新文件，初始化 `records` 数组
4. 更新 `index.json` 中的对应日期记录
5. 回复：`✅ 已记录今日工作（周X）`

### 生成模式（生成周报/本周总结/周报）

#### 搜索范围规则

1. **确定本周起始日期**：
   - 周一为本周第一天
   - 生成周报的当天如果已过18:00，可包含当天；18:00前不包含

2. **搜索本周记录**：
   ```
   本周一 00:00:00 → 今天 23:59:59
   ```
   例如：周四生成周报，搜索范围是 周一00:00 → 周四23:59

3. **按日期升序读取**：从周一到当天，依次读取每天的 `.json` 文件

4. **合并规则**：
   - 同一天的多条记录按 `timestamp` 排序后合并
   - 跨天的相似内容在归纳时自动合并

#### 索引查询流程

1. 读取 `index.json` 获取所有记录日期
2. 筛选出本周日期范围内的记录
3. 按顺序读取对应的 `.json` 文件
4. 合并所有记录内容

---

## 文件操作权限

- 读取：当前用户可读
- 写入：`weekly-records/` 目录及子目录
- 自动创建：日期目录和文件按需创建

---

## 异常处理

| 情况 | 处理方式 |
|------|----------|
| 存储目录不存在 | 自动创建 |
| index.json 损坏/丢失 | 扫描所有子目录重建索引 |
| 单条记录文件丢失 | 跳过该日期，生成周报时标注"部分日期数据缺失" |
| 同一天重复记录 | 保留所有记录，不去重（去重在生成周报归纳时进行） |
